# Step 02：`Version` 结构体（struct + Vec）

> 模块：A 版本模型 ｜ 对应提交：本步提交（commit 信息含 `Lesson: docs/lessons/step-02-version-struct.md`）｜ 演示：✅ 通过（含一条预期警告）｜ 上一步：[Step 01](step-01-scaffold.md)

## 0. 一句话目标
为"版本号"造第一个自定义类型 `Version`，用一个列表装它的各段数字；造两个出来打印看看。

## 1. 前置回顾
Step 01 我们有了能跑的骨架。现在开始建模领域对象。写 Rust 的核心活动，就是**为数据造一个自己的类型**——这一步就造出 `Version`。

## 2. 先写测试（TDD·红）
这一步只是"定义一个类型并打印"，行为还很弱，仍用**运行演示**代替单元测试（下一步 `parse` 起就开始真正的 TDD）。**本步不涉及 TDD 红阶段。**

## 3. 实现到通过（运行演示）
```rust
#[derive(Debug)]
struct Version {
    parts: Vec<u64>,
}

fn main() {
    let a = Version { parts: vec![1, 2, 3] };
    let b = Version { parts: vec![3, 4] };

    println!("a = {:?}", a);
    println!("b = {:?}", b);
}
```
运行：
```
$ cargo run
warning: field `parts` is never read     ← 见 §6，预期内、无害
...
a = Version { parts: [1, 2, 3] }
b = Version { parts: [3, 4] }
```
✅ 两个不同段数的版本都打印出来了。

## 4. 改了哪些文件 / 加了什么
- `src/main.rs`：定义 `struct Version { parts: Vec<u64> }`；在 `main` 里造两个实例并用 `{:?}` 打印。

## 5. 学到的语法 / 技巧
- **`struct`**：定义结构体——把若干相关数据捆成一个有名字的自定义类型。`Version` 是类型名（类型名首字母大写是约定）。
- **字段 `parts: Vec<u64>`**：结构体的一个字段。
  - `Vec` = 可增长的列表；`<u64>` 说明里面装的元素类型；`u64` = 无符号 64 位整数（非负、能存很大）。
  - 所以 `Vec<u64>` = "一串非负整数"。
- **`let a = ...`**：把名字 `a` 绑定到一个值（变量）。变量**默认不可变**，要改得写 `let mut`。
- **`Version { parts: vec![1, 2, 3] }`**：按"字段名: 值"构造一个实例。`vec![...]` 是造列表的**宏**（带 `!`）；这里 `1,2,3` 会被推断为 `u64`（因字段要求 `Vec<u64>`）。
- **`{:?}` 与 `#[derive(Debug)]`**：`{:?}` 用"调试格式"打印；它要求类型具备 `Debug` 能力，而 `#[derive(Debug)]` 让编译器自动生成这套打印代码。

## 6. 语言设计巧思
- **用类型贴合数据的真实形状**：版本段数不定（`3.4` 两段、`1.2.3` 三段），所以字段用**可变长的 `Vec`** 而非写死的几个字段。
- **那条 warning 的含义（warning ≠ error）**：编译**通过、程序照常运行**，只是编译器提醒 `parts` 字段"从没被真正读取过"（Debug 打印在死代码分析里被故意忽略，不算"读"）。Rust 会主动帮你发现"定义了却没用上"的东西。**这条警告是预期内、无害的**——等 Step 04 我们比较版本大小、真正读取 `parts` 时，它会自动消失。

## 7. 领域知识
**版本号的数据形态**：在包管理里，版本号普遍是"由分隔符隔开的一串数字段"。把它建模成"整数列表"是后续做"逐段数字比较"和"约束匹配"的前提。本步只建模、还没赋予比较能力。

## 8. 软件设计理念
- **让数据结构忠实反映领域**：选 `Vec` 而非固定字段，是因为领域里"段数不定"。数据结构选对了，后续逻辑（比较、解析）才顺。
- **单一职责**：`Version` 此刻只负责"持有版本的各段"，不掺别的逻辑。

## 9. 小测验（自测）
1. 为什么 `parts` 用 `Vec<u64>` 而不是固定的三个字段？
2. `u64` 里的 `u` 和 `64` 各是什么意思？
3. `#[derive(Debug)]` 起什么作用？删掉它，`println!("{:?}", a)` 还能编译吗？
4. `field 'parts' is never read` 是错误还是警告？程序还能运行吗？它什么时候会消失？

## 10. 参考答案
1. 版本段数不固定（`3.4` 两段、`1.2.3` 三段），用可变长的 `Vec` 才能装下任意段数；固定字段做不到。
2. `u` = unsigned（无符号，只能非负）；`64` = 64 位（能表示很大的整数）。
3. 它让编译器自动为 `Version` 生成"可调试打印"的代码。删掉后用 `{:?}` 打印会**编译失败**，因为类型不再具备 `Debug` 能力。
4. 是**警告（warning）**，不是错误；程序照常编译运行。它在我们真正"读取" `parts`（Step 04 比较版本时）后消失。
