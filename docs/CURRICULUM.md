# uvr 课程地图（CURRICULUM）

> 方法：自顶向下**理解**全貌，自底向上**建造**。每完成一步就更新本表的进度标记。
> 教学课在 `docs/lessons/`，每步一篇；提交 / PR 中英双语。

## 整体目标
用 Rust 造一个类似 uv / Cargo 的 R 语言包管理器 `uvr`。沿途系统学习 **Rust、R 包管理、软件设计**。

## 模块总览
| 章 | 模块 | 内容 | 状态 |
|---|---|---|---|
| A | 版本模型 | 解析、比较、约束 | ✅ 完成（PR #1） |
| B | 元数据 | DCF 解析、依赖字段、依赖图 | ✅ 完成（PR #2） |
| C | 联网 + 缓存 | HTTP 抓取 PACKAGES、缓存 | ⏸ 延后（需实时网络 / 外部 crate，资源墙） |
| **D** | **依赖求解** ← 当前 | 选版本、传递依赖、冲突、lockfile | 进行中 |
| E | 下载 + 安装 | binary 解压 / `R CMD INSTALL` | ⏸ 延后（需网络 / R） |
| F | 命令行 | 子命令（纯 std 最小实现） | ⬜ |
| G | Benchmark + 报告 | 对 pak 的对照实验 | ⏸ 延后（需 R / pak / hyperfine） |

> 说明：当前沙盒环境下，能**离线、纯逻辑**完成的模块（A、B、D 的求解核心、F 的最小 CLI）优先做完并自合并；
> 需实时网络 / R / 系统安装的模块（C、E、G）留作"资源墙"，待具备条件时再做。
> 模块 D 采用**手写教学版求解器**（非 `pubgrub`）：更利于学习，且不依赖联网拉 crate；`pubgrub` 作为生产升级方向在课中说明。
